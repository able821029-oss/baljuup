# 발주Up — 쉬운 배포 가이드 (왕초보용)

> 인터넷에 우리 앱을 올려서, 어디서든 휴대폰으로 접속할 수 있게 만드는 작업입니다.
>
> **준비물**: 컴퓨터, 이메일 1개, 신용카드 X (전부 무료)
> **소요 시간**: 처음이면 1시간 정도

---

## 큰 그림 — 4단계만 하면 끝

```
[1] GitHub 에 코드 올리기      (= 우리 코드를 인터넷 창고에 보관)
       ↓
[2] Vercel 가입 + 코드 연결    (= 그 창고와 자동 배포 서버 연결)
       ↓
[3] 환경변수 입력              (= 비밀번호 같은 키값을 미리 등록)
       ↓
[4] Deploy 버튼 클릭           (= 끝! 2~3분 뒤 URL 나옴)
```

---

## STEP 1 — GitHub 에 코드 올리기

### 1-1. GitHub 회원가입 (이미 있으면 건너뜀)

1. 브라우저 주소창에 `github.com` 입력
2. 우상단 **Sign up** 클릭
3. 이메일 / 비밀번호 / 사용자명 입력 → 가입 완료

### 1-2. 빈 저장소(repository) 만들기

1. GitHub 로그인 후 우상단 **+** 버튼 → **New repository**
2. 다음과 같이 입력:
   - Repository name: `baljuup` (원하는 이름 가능)
   - Description: (빈칸 OK)
   - **Public** 선택 (Private 이면 Vercel 무료 한도 줄어듦)
   - **❗ 아래 체크박스는 모두 OFF** (README, .gitignore, license 다 안 만듦)
3. **Create repository** 클릭
4. 다음 화면에 나오는 URL 을 메모장에 복사
   - 예: `https://github.com/myname/baljuup.git`

### 1-3. 코드를 GitHub 에 올리기

**Cowork 채팅창에서 저한테 시키면 알아서 합니다.**

저에게 이렇게 말해주세요:
> "GitHub URL `https://github.com/myname/baljuup.git` 에 코드 올려줘"

저는 다음을 해드립니다:
- git init / git add / git commit
- 위 URL 에 push

> ⚠️ 처음이면 GitHub 가 사용자명/비밀번호 대신 **Personal Access Token** 을 요구합니다.
> 그때는 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token (classic) → 권한 `repo` 체크 → 생성된 토큰을 비밀번호 자리에 붙여넣으면 됩니다.

---

## STEP 2 — Vercel 가입 + 프로젝트 만들기

### 2-1. Vercel 가입

1. 주소창에 `vercel.com` 입력
2. **Sign Up** 클릭
3. **Continue with GitHub** 선택 (가장 쉬움)
4. GitHub 권한 요청 화면 → **Authorize Vercel** 클릭
5. Hobby (무료) 플랜 선택

### 2-2. 우리 프로젝트 import

1. Vercel 대시보드에서 **Add New** → **Project** 클릭
2. 화면에 GitHub 저장소 목록이 나옴 → `baljuup` 옆 **Import** 클릭
3. **Configure Project** 화면이 뜸 → 다음 한 곳만 신경쓰면 됩니다:

   ⚠️ **Root Directory** 칸을 보세요
   - 옆에 **Edit** 버튼 클릭
   - 폴더 목록에서 **`waterproof-app`** 선택 → **Continue**

   > ❗ 이걸 안 하면 배포 실패. 우리 코드가 모노레포라 `waterproof-app` 하위에 있어요.

   나머지 (Framework Preset, Build Command 등) 는 **건드리지 마세요**. 자동으로 Next.js 가 잡힙니다.

---

## STEP 3 — 환경변수 입력 (이게 제일 중요)

같은 화면에 **Environment Variables** 섹션이 있습니다. 아래 표를 보면서 하나씩 추가하세요.

### 필수 (이게 없으면 앱이 안 돌아감)

| Key (이름) | Value (값) — 어디서 얻나? |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 대시보드 → Settings → API → Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 같은 화면 → Project API keys → **anon public** |
| `SUPABASE_SERVICE_ROLE_KEY` | 같은 화면 → **service_role** (🔒 절대 노출 금지) |
| `ANTHROPIC_API_KEY` | console.anthropic.com → API Keys → Create Key |
| `DATA_GO_KR_KEY` | data.go.kr 마이페이지 → 인증키 발급 받은 값 |
| `NEXT_PUBLIC_URL` | 일단 `https://placeholder.com` 입력 (배포 후 진짜 URL로 바꿈) |

### 결제 (나중에 유료 시작할 때 입력해도 됨, 지금은 비워도 됩니다)

| Key | Value |
|---|---|
| `NEXT_PUBLIC_TOSS_CLIENT_KEY` | 토스 개발자센터 → 테스트 키 → 클라이언트 키 |
| `TOSS_SECRET_KEY` | 같은 화면 → 시크릿 키 |

### 입력 방법

1. **Key** 칸에 변수 이름 (예: `NEXT_PUBLIC_SUPABASE_URL`)
2. **Value** 칸에 실제 값 (예: `https://abcd1234.supabase.co`)
3. 오른쪽에 환경 선택 — **Production, Preview, Development 모두 체크**
4. **Add** 버튼 클릭
5. 위 표의 모든 변수 반복

---

## STEP 4 — Deploy 버튼 클릭

1. 화면 맨 아래 큰 **Deploy** 버튼 클릭
2. 2~3분 동안 로그가 흘러감 (커피 한 잔 ☕)
3. 🎉 축하 화면 + 우리 앱 URL 발급
   - 예: `https://baljuup-xxx.vercel.app`
4. URL 클릭 → 진짜로 우리 앱이 인터넷에 떠 있는지 확인!

---

## STEP 5 — 배포 후 마무리 (꼭 해야 함)

### 5-1. NEXT_PUBLIC_URL 진짜 값으로 바꾸기

1. Vercel 프로젝트 → **Settings** → **Environment Variables**
2. `NEXT_PUBLIC_URL` 찾아서 **Edit**
3. 값을 진짜 URL (예: `https://baljuup-xxx.vercel.app`) 로 변경 → Save
4. **Deployments** 탭 → 맨 위 항목 우측 **...** → **Redeploy** 클릭

### 5-2. Supabase 에 우리 URL 알려주기 (이메일 인증 작동시키려면 필수)

1. supabase.com 로그인 → 우리 프로젝트 → **Authentication** → **URL Configuration**
2. **Site URL** 칸에 우리 URL 입력 (예: `https://baljuup-xxx.vercel.app`)
3. **Redirect URLs** 에 다음 두 줄 추가:
   ```
   https://baljuup-xxx.vercel.app/auth/callback
   https://baljuup-xxx.vercel.app/**
   ```
4. **Save** 클릭

> 이걸 안 하면 회원가입 후 이메일 인증 링크 눌렀을 때 localhost 로 가서 안 열림.

---

## STEP 6 — 잘 되는지 확인

배포된 URL 에 접속해서 다음 5가지를 순서대로 해보세요:

- [ ] **회원가입 페이지** 가 디자인대로 예쁘게 뜸
- [ ] 가입 시도 → 이메일 인증 메일 도착
- [ ] 메일 인증 링크 클릭 → 대시보드 진입
- [ ] **대시보드** 에 4개 카드 (모니터링 단지 / 즉시 접촉 / 알림 / 수주 성공) 가 보임
- [ ] 휴대폰으로 접속해도 모바일 디자인이 잘 나옴

## 자주 겪는 문제 — 1초 해결법

### 🔴 "Build Failed" 빨간 화면이 떠요
- 십중팔구 **Root Directory** 가 `waterproof-app` 으로 안 잡힌 경우
- Settings → General → Root Directory → `waterproof-app` 으로 수정 → Redeploy

### 🔴 URL 접속하면 "Application error"
- 환경변수 중 뭔가 빠진 경우
- Deployments → 실패한 배포 클릭 → **View Function Logs** 에서 에러 메시지 확인
- 보통 `NEXT_PUBLIC_SUPABASE_URL` 이 없다는 메시지가 자주 나옴

### 🔴 회원가입은 되는데 이메일이 안 와요
- **Supabase → Authentication → Email Templates** 에서 SMTP 설정 필요
- 또는 Supabase 무료 플랜의 기본 SMTP 한도(1시간 4건)에 걸린 것
- Resend / SendGrid 무료 계정 연결 추천 (월 3,000건 무료)

### 🔴 환경변수 추가했는데 적용 안 돼요
- **Redeploy 필수**: Deployments → ... → Redeploy
- 환경변수는 빌드 시점에만 주입되므로 재배포 없이는 반영 안 됨
