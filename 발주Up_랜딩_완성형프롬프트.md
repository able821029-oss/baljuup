# 발주Up 랜딩페이지 v3 — 완성형 재구축 프롬프트

---

## 📋 작업 개요

`landing_final.html`을 기반으로, 심리마케팅 원칙과 타겟 사용자 특성에 맞게
**전환 최적화된 단일 HTML 파일**로 완전히 재구축한다.

디자인 시스템(네이비 다크 테마, CSS 변수, Noto Sans KR)과 핵심 메시지
"공고 전에 먼저 연락하면 경쟁자가 없다"는 유지한다.

---

## 🎯 타겟 사용자 프로필 (이 파일을 코딩 내내 머릿속에 유지할 것)

**인구통계**
- 40~60대 남성, 수도권 방수·방수공사 영업담당자 또는 소규모 사업주
- 스마트폰 사용에 익숙하나 웹 결제·SaaS 구독 경험은 적음
- 카카오톡·전화 중심의 비즈니스 문화

**심리적 특성**
- 의사결정 방식: 충동 결제 없음, 신뢰 → 전화 → 검토 → 결정 순서
- 핵심 공포: "경쟁사가 나보다 먼저 연락했을 것이다"
- 핵심 욕구: 안정적 수주 파이프라인, 영업 낭비 시간 감소
- 정보 수용 방식: 긴 글 기피, 핵심만 빠르게 훑음, 숫자보다 상황 공감
- 기술 신뢰 임계점: 수치가 과하면 "광고 같다"고 이탈

## 🧠 적용할 심리마케팅 원칙 (섹션별로 명시함)

| 원칙 | 한국어 | 적용 방식 |
|------|--------|-----------|
| Loss Aversion | 손실 회피 | "공고 뜨면 이미 3곳이 전화하고 있다" |
| Gain Framing | 이득 프레이밍 | "먼저 전화한 쪽이 수주한다" |
| Authority | 권위 | 국토교통부 공개 데이터 기반, K-apt 연계 |
| Scarcity | 희소성 | 베타 선착순, 수도권 한정 |
| Reciprocity | 호혜성 | 무료 체험, 카드 불필요, 부담 없음 |
| Social Proof | 사회적 증명 | 현실적 업무 시나리오 (scripted 느낌 금지) |
| Familiarity | 친숙함 | 카카오톡 알림, 아침 루틴, 익숙한 단지명 |
| Simplicity | 단순성 | 첫 화면 3가지만: 무엇을·왜·어떻게 |

## 🚫 절대 하지 말아야 할 것 (기존 랜딩의 실패 요인)

1. **요금제 섹션 금지** → 베타 신청 섹션으로 대체
2. **퍼센트 수치 남발 금지** (94%, 88%, 73%)
   → "발주 임박", "관심 단지", "검토 단계" 정성 레이블로 대체
3. **"실시간" 3회 초과 사용 금지** → "매일 오전 업데이트"로 대체
4. **지도 섹션 완전 삭제** → 지역별 건수 카드로 대체
5. **과장형 카카오톡 대화 금지** → 실무적·현실적 톤으로 교체
6. **127건, 17천+, 1,247개 등 미검증 숫자 금지** → 정성적 표현 사용
7. **ticker(뉴스 띠) 삭제** → 신뢰 저하 요소
8. **완성형 SaaS 느낌 금지** → 베타, 진솔함, 실제로 쓸 수 있을 것 같은 느낌

### 6. VS 섹션 — K-apt vs 발주Up

**심리 원칙: 권위 + 명확한 차별성**

섹션 태그: `핵심 차이`
H2: `딱 한 줄로 비교합니다`

```
K-apt                           발주Up
─────────────────               ─────────────────
✗ 공고가 올라오면 확인          ✓ 공고 3~6개월 전에 파악
✗ 경쟁사와 동시에 알게 됨       ✓ 경쟁사보다 먼저 연락
✗ 입찰까지 14일 남음           ✓ 여유 있게 관계 형성
✗ 제안서 직접 만들어야 함       ✓ 클릭 한 번 — 제안서 완성
✗ 누가 담당인지 모름           ✓ 관리소 연락처 제공
```

하단 보조 문구:
`K-apt는 확인용, 발주Up은 영업용입니다.`
