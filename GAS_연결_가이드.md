# 사전모집 폼 → 구글 시트 연결 (10분 완료)

> 결과: 랜딩페이지 v2에서 누군가 전화번호 입력하면
> → 자동으로 구글 시트에 한 줄 추가됨
>
> 필요한 것: 구글 계정 1개. 끝.

---

## STEP 1 — 구글 시트 만들기 (1분)

### 1-1. 새 시트 생성
1. 브라우저에서 https://sheets.google.com 접속
2. 본인 구글 계정 로그인
3. **+ 새 스프레드시트** 클릭 (큰 +)
4. 시트 이름 변경: 좌상단 "제목없는 스프레드시트" → **`발주Up 사전등록`** 입력

### 1-2. 시트 ID 복사
주소창의 URL 을 보면:
```
https://docs.google.com/spreadsheets/d/[이 부분이 ID]/edit#gid=0
```

`/d/` 와 `/edit` **사이 문자열을 복사** (보통 44자 정도).

[메모] 메모장에 임시 저장 — STEP 2에서 씁니다.

---

## STEP 2 — Google Apps Script 설정 (3분)

### 2-1. Apps Script 열기
1. 새 탭에서 https://script.google.com 접속
2. **+ 새 프로젝트** 클릭
3. 좌상단 "제목없는 프로젝트" → **`발주Up 등록 API`** 로 이름 변경

### 2-2. 코드 붙여넣기
1. 가운데 코드 에디터의 모든 내용 **전체 삭제** (Ctrl+A → Delete)
2. `E:\01.앱 개발\방수 SaaS MVP\apps_script.gs` 파일을 메모장으로 열기
3. 전체 복사 (Ctrl+A → Ctrl+C) → Apps Script 에디터에 붙여넣기 (Ctrl+V)

### 2-3. 시트 ID 교체
코드 20번째 줄 찾기:
```javascript
const SPREADSHEET_ID = 'REPLACE_WITH_YOUR_SHEET_ID';
```

**`REPLACE_WITH_YOUR_SHEET_ID`** 만 메모장의 시트 ID 로 교체.

(작은 따옴표는 그대로 유지)

### 2-4. 저장
- **Ctrl+S** (또는 좌상단 디스크 아이콘) → 저장

---

## STEP 3 — 권한 부여 + 배포 (3분)

### 3-1. 첫 권한 부여 (한 번만)
1. 상단의 **함수 선택** 드롭다운에서 **`testInsert`** 선택
2. **▶ 실행** 버튼 클릭
3. "권한 검토 필요" 팝업 → **권한 검토** 클릭
4. 본인 구글 계정 선택
5. "Google에서 확인하지 않은 앱" 경고 → **고급** 클릭 → **`발주Up 등록 API`(으)로 이동(안전하지 않음)** 클릭
6. **허용** 클릭

3-2 으로 진행하기 전 시트로 가서 확인:
- 위에서 만든 `발주Up 사전등록` 시트에 **`신청자`** 탭이 자동 생성됨
- 한 줄 데이터가 추가됨 (테스트 방수 / 01099998888)
- 이게 보이면 권한 + 코드 정상

### 3-2. 웹앱 배포
1. Apps Script 화면 우상단 **배포** 클릭
2. **새 배포** 클릭
3. (톱니바퀴) → **웹 앱** 선택
4. 설정:
   - **설명**: `발주Up 사전등록 v2`
   - **다음 사용자로 실행**: `나(본인 이메일)`
   - **액세스 권한이 있는 사용자**: **`모든 사용자`** ← 중요!
5. **배포** 클릭
6. 다시 "권한 검토" 뜨면 위와 같이 허용

### 3-3. URL 복사 (가장 중요!)
배포 완료 화면에서:
- **웹 앱 URL** 항목에 `https://script.google.com/macros/s/AKfycb...../exec` 형태의 URL
- 우측 **복사** 버튼 클릭

[메모] 메모장에 저장 — STEP 4에서 씁니다.

---

## STEP 4 — 랜딩페이지에 URL 입력 (1분)

### 4-1. 파일 수정
1. `E:\01.앱 개발\방수 SaaS MVP\landing_v2_사전모집.html` 파일을 메모장으로 열기
2. **Ctrl+F** 로 검색: `REPLACE_WITH_YOUR_DEPLOYMENT_ID`
3. 찾은 줄:
   ```javascript
   const GAS_ENDPOINT = 'https://script.google.com/macros/s/REPLACE_WITH_YOUR_DEPLOYMENT_ID/exec';
   ```
4. **`REPLACE_WITH_YOUR_DEPLOYMENT_ID`** 부분만 STEP 3-3에서 복사한 URL 의 `s/` 와 `/exec` 사이 문자열로 교체

또는 더 간단히: **작은따옴표 안의 전체 URL** 을 STEP 3-3 URL 로 통째로 교체.

5. **Ctrl+S** 저장

### 4-2. 테스트 (로컬)
1. 파일을 더블클릭으로 브라우저에서 열기 (또는 우클릭 → 연결 프로그램 → Chrome)
2. 휴대폰 번호 `01099998888` 입력 → **등록하기**
3. "등록 완료!" 모달이 뜨면 성공
4. 구글 시트 새로고침 → 새 줄이 추가됐는지 확인

## STEP 5 — 라이브에 배포 (선택)

### 옵션 A: Vercel 별도 배포 (추천)
랜딩페이지를 본 앱과 분리해서 광고용으로:

1. https://vercel.com 접속 → 우상단 **Add New → Project**
2. **Import Git Repository** 또는 **Upload Files** 선택
3. landing_v2_사전모집.html 파일만 업로드 (또는 별도 git repo)
4. 배포 후 도메인을 `welcome.baljuup.co.kr` 같이 설정 가능

### 옵션 B: 기존 앱에 /welcome 경로로
복잡해서 Claude Code 에 맡기는 게 좋습니다:

```
Claude Code 에 전달:
"landing_v2_사전모집.html 을 baljuup.vercel.app/welcome 경로에서
서빙되도록 next.js public 폴더 또는 별도 페이지로 추가해주세요."
```

### 옵션 C: 임시 — Notion/카카오톡으로 파일 직접 공유
SNS에 직접 HTML 파일 공유는 안 됩니다. Vercel 또는 다른 호스팅 필수.

## 자주 발생하는 문제

### [에러] "등록 중..." 으로 멈춤
**원인**: GAS_ENDPOINT URL 이 잘못 입력됨
**해결**:
1. URL 끝이 `/exec` 으로 끝나는지 확인
2. 큰따옴표/작은따옴표 둘러쌓여 있는지 확인
3. 배포된 GAS 가 "모든 사용자" 액세스로 설정됐는지 확인

### [에러] 시트에 데이터 안 들어옴
**원인**: SPREADSHEET_ID 가 잘못됨
**해결**: Apps Script 에디터로 가서 `testInsert` 함수 다시 실행 → 에러 메시지 확인

### [참고] CORS 에러 (개발자도구)
**정상**: GAS 는 응답을 못 받지만 데이터는 들어갑니다. (no-cors 모드라 정상)

## 매번 새 코드 배포 시 주의

GAS 의 코드를 수정하면:
1. **저장만 하면 안 되고 "배포 → 배포 관리 → (편집) → 새 버전" 으로 재배포 필수**
2. 배포 후 URL 은 **그대로 유지** (버전만 올라감)
